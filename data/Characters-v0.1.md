# 角色人物设计文档（初版 v0.1）

> 本文档用于定义《Dominion：多元世界地产议会》中的**初版可选角色人物**。
> 设计目标是：**每个角色代表一种明确的玩法风格（Playstyle）**，而不是单纯数值强弱。

--- 

## 1. 设计原则

- 玩法导向：角色 = 策略选择，而非剧情主角  
- 数值对称：所有角色总属性点数一致，仅分布不同  
- 世界观绑定：角色身份与“议会 / 多元世界”背景一致  
- 多人博弈友好：角色能力可相互克制，避免单一最优解  

---

## 2. 通用属性定义

| 属性 | 说明 |
|----|----|
| 💰 资本（Capital） | 初始资金、现金流能力 |
| 🍀 幸运（Luck） | 正负事件触发概率修正 |
| 🤝 谈判（Negotiation） | 买地、交易、赎回、拍卖折扣 |
| 🎭 魅力（Charisma） | 联盟、投票、政治事件影响 |
| 🧠 科技（Tech） | 星球地图、高级建筑、特殊规则 |
| 🔋 行动力（Stamina） | 连续行动、额外回合、恢复速度 |

- 属性范围：1 ~ 10  
- 初版平衡原则：每名角色属性点数总和相同

## 4. 后续扩展方向

- 每名角色增加专属事件卡（3–5 张）
- 引入成长 / 退化机制
- 角色与地图产生专属联动规则
- 支持 MOD：玩家自定义角色模板
